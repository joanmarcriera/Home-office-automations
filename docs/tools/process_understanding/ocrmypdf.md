# OCRmyPDF

## What it is
OCRmyPDF adds an OCR (Optical Character Recognition) text layer to scanned PDF files, allowing them to be searched and selected.

## What problem it solves
It fixes "image-only" PDFs that were generated by scanners, making them accessible and searchable. It also optimizes PDF size and cleans up artifacts.

## Where it fits in the pipeline
**Process**

## Typical use cases (in this homelab / family automation context)
- **Archiving Old Scans**: Processing years of old paper records that were scanned as images.
- **Searchable Manuals**: Converting scanned appliance manuals into searchable documents.
- **Automated Intake**: Part of a pipeline where incoming files are automatically OCRed before being stored in Nextcloud or Paperless.

## Integration points
- **Paperless-ngx**: Integrated by default as the primary OCR engine.
- **n8n**: Can be called as a shell command in an automation workflow.
- **Local Scripts**: Easily scriptable for batch processing of directories.

## Strengths
- Extremely reliable and handles complex layouts well.
- Generates standard, high-quality PDF/A files.
- Includes advanced features like deskewing and page rotation.

## Limitations
- Command-line only (no native GUI).
- Resource intensive for large batch jobs.
