    AI Company Starter Stack¶

    This page turns the catalogue into an opinionated starter stack for building a company where AI is part of daily operations, not just a side experiment.

    The goal is not to list every good tool. The goal is to help you choose the smallest stack that gives you leverage across product, operations, research, content, and internal execution.

    The default stack¶

    Layer Default choice Why it belongs in the starter stack Replace when
    Website / app surface Free AI Website Playbook -> Vercel + Supabase Gives you a practical default for public sites, MVPs, and internal tools Your architecture has stronger static-site or infra constraints
    Agent operating model Claude Skills Ecosystem + Superpowers Reusable workflows plus execution discipline You are not using coding agents seriously yet
    Current technical context Context7 Keeps coding agents grounded in current docs Your work is mostly non-technical or repo-local
    Workflow control plane n8n Scheduling, retries, approvals, and cross-system automation Your company is still too small for durable workflows
    Web interaction Browser Use Covers UI-only systems and interactive web tasks Stable APIs exist for the same work
    Company operating surface Google Workspace CLI Documents, calendars, files, and admin actions become automatable You do not run the company on Google Workspace
    Memory mem0 Preserves account, process, and user context across sessions Tasks are stateless and one-off
    Research harness DeerFlow Strong fit for strategy, research, and evidence synthesis Your work is mostly transactional automation
    Claude implementation examples Claude Cookbooks First-party examples reduce integration guesswork You already have strong internal patterns
    Local/private inference option LocalAI + llmfit Gives you a path to self-hosting without blind hardware bets Frontier cloud models are more important than control

    Expansion packs¶

    Pack Add these tools Use when Main output
    Website launch pack Free AI Website Playbook + Vercel + Supabase You need a public launch surface, waitlist, or early MVP Public web presence and product shell
    Knowledge workspace pack AnythingLLM + LocalAI or Ollama Teams need a shared internal assistant over documents and knowledge Internal AI workspace
    Routing and model control pack LiteLLM + OpenRouter + ClawRouter You need cost, fallback, or provider-routing control across many agents Model governance and routing
    Research and lead-intel pack DeerFlow + Tavily + Browser Use + mem0 Sales, strategy, or market research becomes a core workflow Reusable intelligence pipelines
    Finance and market pack OpenBB + n8n + Google Workspace CLI Finance, investor, or market reporting needs become recurring Structured finance briefings
    Visual builder pack Flowise + Supabase You want non-engineers or mixed teams to ship internal AI tools faster Prototype-to-internal-app path

    What each part is for¶

    0. Website layer¶

    Use this when the company needs an actual web surface, not only internal automation.

    • The Free AI Website Playbook helps choose what kind of site to build and what host fits.
    • Vercel is the default when speed of launch matters.
    • Supabase is the default backend once forms, auth, or product state appear.
    • Use GitHub Pages for docs-heavy sites and Cloudflare Pages for static directories and static-first public properties.

    1. Skills + Superpowers¶

    Use this pair when you want agents to behave like trained operators instead of smart autocomplete.

    • Skills package repeatable company procedures.
    • Superpowers enforces a better process for execution.
    • Together, they make AI usable by teams, not only by one power user.

    2. n8n + Workspace CLI¶

    Use this pair when the company needs actual operating workflows.

    • n8n coordinates timing, branching, retries, approvals, and logs.
    • Google Workspace CLI executes useful work in Docs, Sheets, Drive, Calendar, and Chat.
    • Together, they turn office operations into an automatable system.

    3. Context7 + Claude Cookbooks¶

    Use this pair when the team is building AI products or internal tools.

    • Context7 provides current third-party docs.
    • Claude Cookbooks provides first-party Claude implementation patterns.
    • Together, they reduce bad assumptions during build work.

    4. Browser Use + mem0 + DeerFlow¶

    Use this trio for research, lead generation, and web-native execution.

    • Browser Use interacts with websites.
    • mem0 remembers what happened and what matters.
    • DeerFlow structures longer research/execution tasks.

    5. LocalAI + llmfit¶

    Use this pair when privacy, local control, or cost discipline matters.

    • llmfit decides what can run on your hardware.
    • LocalAI gives you a reusable local API once you know the hardware plan is viable.

    6. Routing layer: LiteLLM vs OpenRouter vs ClawRouter¶

    Use this layer when model choice, fallback behavior, or provider economics become operational concerns.

    • LiteLLM is the best general internal routing/control plane.
    • OpenRouter is the fastest way to access many hosted models behind one API and billing layer.
    • ClawRouter is the specialized choice when OpenClaw-style agent routing is part of the architecture itself.

    7. Knowledge workspace: AnythingLLM¶

    Use this when the company needs a usable AI workspace for teams, not just back-end components.

    • AnythingLLM is a fast way to stand up internal knowledge assistants.
    • It is especially useful when the team wants document-grounded chat before investing in custom product work.

    8. Finance intelligence: OpenBB¶

    Use this when structured financial or market intelligence should feed founder, investor, or sales workflows.

    • OpenBB adds structured market data that generic web search cannot replace.
    • It is most useful for companies where market, macro, or financial context changes decisions.

    Comparison table¶

    Need Best default Use instead when Comment
    Public website or product shell Free AI Website Playbook + Vercel Static docs or content site is enough The playbook is the decision layer; Vercel is the common default
    Reusable company procedures Claude Skills Ecosystem You only need one-off prompts Skills matter once the process repeats
    Reliable coding-agent execution Superpowers Speed matters more than rigor Best for important engineering work
    Workflow orchestration n8n One small script is enough n8n is the operating system, not just a node editor
    Browser-only workflows Browser Use API exists APIs beat browsers when available
    Workspace automation Google Workspace CLI Your company runs elsewhere Great when Docs/Sheets/Drive are core
    Long-lived memory mem0 Stateless execution is enough Do not add memory unless it pays for itself
    Deep research DeerFlow Search + one summary call is enough Best for evidence-heavy work
    Current docs for agents Context7 No external docs required Very high leverage for coding teams
    First-party Claude patterns Claude Cookbooks You need broader ecosystem docs Strong complement to Context7
    Local inference LocalAI You need frontier quality and speed first Pair with llmfit before buying hardware
    Internal AI workspace AnythingLLM You need full custom product behavior Great for shared internal assistants
    Model routing LiteLLM You only need one hosted provider Use OpenRouter for simpler hosted access; use ClawRouter for OpenClaw-heavy stacks
    Finance / market intelligence OpenBB Generic web research is enough Best when market data should be structured and recurring

    Example starter stacks¶

    Lean AI-native services company¶

    • Free AI Website Playbook
    • Vercel
    • Supabase
    • Claude Skills Ecosystem
    • Superpowers
    • Context7
    • n8n
    • Google Workspace CLI

    Use this when you want the smallest serious stack for delivery, documentation, and operations.

    Research-heavy AI consultancy¶

    • DeerFlow
    • Browser Use
    • Tavily
    • mem0
    • n8n

    Use this when account research, competitor analysis, and evidence gathering are core to the business.

    Privacy-first internal automation stack¶

    • LocalAI
    • llmfit
    • Ollama
    • n8n
    • Supabase

    Use this when local control and internal data handling matter more than frontier-model convenience.

    Shared internal knowledge stack¶

    • AnythingLLM
    • LocalAI
    • Ollama
    • mem0
    • Google Workspace CLI

    Use this when the company needs an internal assistant surface that people can actually use daily.

    Multi-model routing stack¶

    • LiteLLM
    • OpenRouter
    • ClawRouter
    • Context7

    Use this when many models, providers, or routing policies are becoming a cost and reliability problem.

    Finance and market intelligence stack¶

    • OpenBB
    • DeerFlow
    • Tavily
    • n8n

    Use this when finance, market context, or target-account intelligence needs to become systematic.

    What I would use first¶

    If I were setting up an AI-driven company from scratch, I would start with:

    1. Claude Skills Ecosystem
    2. Superpowers
    3. Context7
    4. n8n
    5. Google Workspace CLI

    Then I would add:

    • Browser Use when APIs are missing
    • mem0 when continuity actually matters
    • DeerFlow when research becomes a core workflow
    • LocalAI and llmfit when local/private inference is justified
    • AnythingLLM when teams need a shared internal AI workspace
    • LiteLLM or OpenRouter when model routing becomes operationally important
    • OpenBB when finance and market intelligence become recurring workflows
